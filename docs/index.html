GoToPoint Module

Publishes to:

/cmd_vel

ActionServer:

/go_to_point

Description:

Using an action server, this node controls a non-holonomic robot to produce a go_to_point behaviour. An FSM is used to model the behavior whenever a new goal pose is received.

scripts.go_to_point.go_to_point(req)[source]

Set the appropriate behaviour depending on the current robot state, in order to reach the goal.

The state machine keeps running until the goal is reached or the action is preempted (the goal gets cancelled).

Parameters

req – (x,y,theta) goal pose
